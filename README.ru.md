# yadisk-dump

Скачивает все файлы с Яндекс.Диска в локальную папку. Только чтение, с продолжением после остановки.

## Быстрый старт

1. Установите:

   ```bash
   pipx install git+https://github.com/fomin-n/yadisk-dump
   ```

2. Запустите:

   ```bash
   yadisk-dump
   ```

3. Вставьте OAuth-токен по запросу. После нажатия Enter инструмент откроет нужную страницу Яндекса.

## Команды

| Команда | Поведение |
|---|---|
| `yadisk-dump` | Мастер при необходимости, сканирование, подтверждение и полное скачивание |
| `yadisk-dump pull [--to DIR] [--workers N] [--yes]` | Неинтерактивное сканирование и скачивание для скриптов и cron |
| `yadisk-dump scan` | Только сканирование и итоговая таблица |
| `yadisk-dump status` | Сохраненные счетчики и сведения о последнем запуске |
| `yadisk-dump retry` | Повторное скачивание файлов со статусом `failed` |
| `yadisk-dump verify` | Проверка локальных MD5; несовпадения становятся повторяемыми ошибками |
| `yadisk-dump logout` | Удаление сохраненного файла учетных данных |

Глобальные параметры: `--to DIR`, `--workers N` (ограничивается диапазоном 1–5), `--yes`, `--quiet` и `--version`. По умолчанию используется `~/YandexDisk`; интерактивный режим запоминает другой выбор.

## Безопасность — куда уходят данные

Сетевая поверхность работающей программы намеренно мала:

- `GET https://cloud-api.yandex.net/v1/disk` проверяет учетные данные и читает квоту.
- `GET https://cloud-api.yandex.net/v1/disk/resources` перечисляет каталоги и файлы.
- `GET https://cloud-api.yandex.net/v1/disk/resources/download` запрашивает одну временную подписанную ссылку непосредственно перед файлом.
- Обычный потоковый `GET` читает подписанную ссылку с хоста скачивания Яндекса, возвращенного API. OAuth-токен не прикладывается. Перенаправления на другой хост запрещены.
- OAuth-полигон открывается в обычном браузере только после нажатия Enter. Установка пакета может обращаться к PyPI; работающая CLI-программа — нет.

Наследование прокси и `.netrc` отключено, поэтому байты нельзя незаметно направить через настроенный сторонний прокси. Нет телеметрии, аналитики, отчетов о сбоях, проверки версий, автообновления и исходящей загрузки файлов.

Удаленная сторона доступна только для чтения по конструкции. Обертка [`src/yadisk_dump/api.py`](src/yadisk_dump/api.py) лишь проверяет учетные данные, читает сведения о Диске, перечисляет каталоги и запрашивает ссылки. В ней нет вызовов загрузки, удаления, перемещения, публикации, создания удаленных каталогов или других изменяющих операций.

OAuth-токен берется из `YADISK_TOKEN` или локального файла:

- POSIX: `${XDG_CONFIG_HOME:-~/.config}/yadisk-dump/token`, режим `0600`.
- Windows: `%APPDATA%\yadisk-dump\token`.

Он никогда не печатается, не записывается в лог или причину ошибки и не передается хосту скачивания. Подписанные URL также не попадают в ошибки.

Каждый компонент удаленного пути очищается. Обход каталогов, выход через симлинк, зарезервированные имена Windows, запрещенные символы, нормализованные/регистронезависимые коллизии и длинные пути Windows обрабатываются до открытия файла. Данные пишутся в `<имя>.part`, проверяются по размеру, сбрасываются на диск и атомарно заменяют итоговый файл. SQLite-состояние находится в `<каталог>/.yadisk-dump/state.db` и работает в WAL-режиме.

Проверка статических URL:

```bash
grep -rnE "https?://" src/
```

Результат содержит только `cloud-api.yandex.net` и OAuth-полигон. Зависимости времени выполнения ограничены `yadisk` и `rich`; `requirements.lock` фиксирует полное разрешенное дерево.

## Частые вопросы

### Сколько действует токен?

Срок задает Яндекс; обычно токен действует около года. После истечения снова запустите `yadisk-dump` и вставьте новый. Некорректный `YADISK_TOKEN` нужно заменить или удалить из окружения: он имеет приоритет над файлом.

### Что происходит при HTTP 429?

Программа соблюдает `Retry-After`, а без него ждет 60 секунд. Ожидания из-за лимита не расходуют пять обычных попыток.

### Можно продолжить после Ctrl+C или сбоя?

Да. Уже существующие файлы правильного размера пропускаются, незавершенные `.part` удаляются, ошибки сохраняются в SQLite. `yadisk-dump retry` обрабатывает их напрямую. Если файл исчез удаленно, локальная копия никогда не удаляется.

### Что делает проверка?

`yadisk-dump verify` вычисляет хеш завершенных локальных файлов и сравнивает его с MD5 последнего API-сканирования. Отсутствующие или отличающиеся файлы получают статус `failed`, после чего `yadisk-dump retry` скачивает их заново. Обычная загрузка проверяет размеры и не хеширует каждый существующий файл.

### Поддерживаются длинные пути Windows?

Да. Компоненты делаются совместимыми с Windows, а абсолютные пути длиннее примерно 240 символов используют расширенную форму `\\?\`, включая UNC.

### Почему нет «безлимитного хранилища фото»?

Оно требует браузерных cookie и недокументированных Photoslice API. Такой режим намеренно исключен. Инструмент использует только официальный [REST API Яндекс.Диска](https://yandex.ru/dev/disk/rest?lang=ru).
